# Shopify練習アプリ 要件定義書（初版）

作成日: 2026-06-07
対象: アパレル系職種向け（EC担当、MD、バイヤー、商品登録担当、在庫管理担当）

---

## 1. 背景・目的

アパレル業務では、SKUが多く、サイズ・カラー別の在庫管理、シーズン/カテゴリ別の陳列、ブランド訴求ページの更新が日常的に発生する。
本システムは、Shopifyで実務上よく行う運用（商品登録、在庫管理、コレクション管理、ブランドページ管理）を、学習・練習できるWebアプリとして提供することを目的とする。

また、単なるモックではなく、可能な範囲でShopifyの実機能・概念に沿った体験を提供し、将来的にはShopifyストア連携にも拡張できる設計とする。

---

## 2. Shopify調査結果サマリ

以下は、Shopify公式ドキュメント・ヘルプをもとに整理した、本件に関係する主要機能。

### 2.1 商品管理
- Shopifyでは商品（Product）を管理できる
- 商品にはタイトル、説明、画像、価格、商品タイプ、ベンダー（Vendor/ブランド相当）、タグ等を保持できる
- バリエーション（Variant）により、サイズ・カラーなどの違いを管理できる
- アパレル用途では「1商品 = 親商品」「サイズ/カラー = バリエーション」という運用が基本

### 2.2 在庫管理
- 在庫はバリエーション単位で管理できる
- Shopifyは在庫追跡機能を持ち、在庫数・在庫切れ状態の管理が可能
- 複数ロケーション前提の在庫管理にも対応している
- 実務では、サイズ別・カラー別SKUごとの在庫更新が重要

### 2.3 コレクション管理
- 商品をコレクションで分類できる
- 手動コレクション（manual collection）と自動コレクション（smart/automated collection）の考え方がある
- 例: 「2026SS」「レディース」「アウター」「セール対象」など
- 商品の発見性向上や販促導線設計に有効

### 2.4 ブランドページ・独自コンテンツ管理
- Shopify標準項目だけでは、ブランド紹介文、ブランド画像、ブランドストーリー等の表現が不足する場合がある
- これを補う仕組みとしてMetafields（個別拡張項目）とMetaobjects（構造化された独自データ）が利用できる
- Metaobjectsはブランド情報のような「複数項目から成る独立データ」の管理に向く
- テーマ側では、動的ソースやLiquidを用いて、ブランドページへデータ表示できる

### 2.5 本件への示唆
- 商品登録: Shopifyの基本機能に適合
- 在庫管理: バリエーション/SKU単位で再現可能
- コレクション: Shopifyの主要運用に適合
- ブランドページへのデータ注入: Metaobjects/Metafieldsを活用する設計が最も自然

## 3. システム化方針

### 3.1 システムの位置づけ
本アプリは、アパレル業務担当者がShopify運用を学ぶための「練習用Webアプリ」とする。

### 3.2 方針
- 初期フェーズでは、学習しやすい独自UIを提供する
- データ構造・用語はShopifyに寄せる
- 将来的にShopify開発ストアやテストストアとAPI連携可能な設計にする
- ブランドページ情報は、ブランドマスタとして管理し、商品やコレクションと関連付ける

## 4. 対象ユーザー

### 4.1 想定ユーザー
- EC運用担当
- 商品登録担当
- 在庫管理担当
- MD（マーチャンダイザー）
- ブランド担当/販促担当

### 4.2 ユーザー課題
- Shopifyの概念（商品/バリエーション/SKU/コレクション）がわかりにくい
- アパレル特有のサイズ・カラー管理に慣れる必要がある
- ブランド訴求ページへの情報反映の流れを理解したい
- 実ストアを壊さずに練習したい

## 6. 業務シナリオ

### シナリオ1: 商品登録担当
1. 新商品を登録する
2. ブランドを選択する
3. カラー・サイズを設定する
4. SKUを自動生成する
5. 画像と説明文を登録する
6. 公開状態を下書き/公開で切り替える

### シナリオ2: 在庫担当
1. 商品を検索する
2. SKUごとの在庫数を入力する
3. ロケーション別在庫を更新する
4. 在庫切れ商品を確認する

### シナリオ3: EC/MD担当
1. 商品をコレクションに追加する
2. シーズン/カテゴリ/セールなどで絞り込む
3. 自動分類ルールを設定する

### シナリオ4: ブランド担当
1. ブランド情報を登録する
2. ブランドロゴ、メインビジュアル、紹介文を入力する
3. ブランドストーリー、掲載順、関連コレクションを設定する
4. ブランドページ表示イメージを確認する

## 7. 機能要件

### 7.1 商品管理

#### 7.1.1 商品登録
- 商品名
- 商品説明
- ブランド
- 商品タイプ
- 性別カテゴリ
- シーズン
- 販売価格
- 比較価格
- 公開状態（下書き/公開）
- 商品画像複数登録
- タグ設定

#### 7.1.2 バリエーション管理
- サイズ（XS/S/M/L/XL 等）
- カラー（Black/White/Navy 等）
- サイズ×カラーの組み合わせ生成
- SKU自動採番
- バーコード任意入力
- バリエーションごとの価格差分設定（任意）

#### 7.1.3 商品一覧
- 一覧表示
- キーワード検索
- ブランド絞り込み
- コレクション絞り込み
- 公開状態絞り込み
- 在庫有無絞り込み

### 7.2 在庫管理

#### 7.2.1 在庫更新
- SKU単位で在庫数登録
- 一括更新（CSV形式相当のUIでも可）
- 入荷/出荷/調整の区分を記録

#### 7.2.2 在庫表示
- 商品別在庫一覧
- バリエーション別在庫一覧
- 在庫切れ一覧
- 在庫少一覧（しきい値設定）

#### 7.2.3 ロケーション管理（初期は簡易版）
- 倉庫
- 店舗
- EC共通在庫

### 7.3 コレクション管理

#### 7.3.1 手動コレクション
- コレクション作成
- タイトル
- 説明文
- 対象商品の手動追加/削除

#### 7.3.2 自動コレクション（簡易ルール）
- ブランド = ○○
- タグに○○を含む
- 商品タイプ = ○○
- 在庫あり
- シーズン = ○○

#### 7.3.3 コレクション一覧
- 商品数表示
- 公開状態表示
- 並び順設定

### 7.4 ブランド管理

#### 7.4.1 ブランドマスタ
- ブランド名
- ブランドコード
- ロゴ画像
- ブランドメイン画像
- 紹介文
- ブランドストーリー
- ブランドコンセプト
- 公開状態
- 表示順

#### 7.4.2 ブランドページ用データ
- ヒーローバナー画像
- キャッチコピー
- ブランド詳細説明
- 関連商品一覧
- 関連コレクション一覧
- SNSリンク
- 外部URL

#### 7.4.3 データ注入イメージ
- ShopifyでいうMetaobjects相当のブランドオブジェクトを持つ
- 商品はブランドオブジェクトを参照する
- ブランドページはブランドオブジェクトのデータを描画する
- 将来的にはShopifyのMetafields/Metaobjectsへマッピング可能にする

### 7.5 学習支援機能
- サンプルブランド・サンプル商品を初期投入
- チュートリアル表示
- 実務シナリオ別の演習モード
- 入力エラー時のガイド
- 用語説明（商品、バリエーション、SKU、コレクション等）

### 7.6 操作履歴
- 誰がいつ何を登録/更新したか記録
- 学習結果の確認
- 誤操作のトレース

## 10. Shopify連携方針（将来拡張）

### 10.1 連携対象
- Product
- Variant
- Inventory
- Collection
- Metafields
- Metaobjects

### 10.2 連携方式
- Shopify Admin API を利用した同期
- 練習モードではローカルDB保持
- 連携モードではShopifyへPush/ShopifyからPull

### 10.3 マッピング方針
- Product → Shopify Product
- Variant → Shopify Variant
- Inventory → Shopify Inventory
- Collection → Shopify Collection
- Brand/BrandPageContent → Shopify Metaobject + Metafield参照

## 11. 非機能要件

### 11.1 性能
- 商品一覧初期表示: 3秒以内
- 商品登録保存: 2秒以内
- 在庫一括更新: 100SKUで5秒以内を目安

### 11.2 セキュリティ
- ログイン認証
- 操作権限の基本分離（閲覧/編集/管理）
- 操作ログ保存

### 11.3 可用性
- 学習用途のため高可用性は必須ではないが、平日日中の安定稼働を想定

### 11.4 保守性
- Shopify連携部を疎結合化
- 商品/在庫/ブランドをモジュール分離

## 14. 想定リスク

- Shopifyテーマごとにブランドページ実装方法が異なる
- ブランドページは標準機能だけでは表現不足となるため、Metaobjects前提設計が望ましい
- 在庫の実務運用はロケーションや出荷ルールで複雑化しやすい
- 練習用UIとShopify実画面の差が大きいと学習効果が下がる

---

## 15. 推奨アーキテクチャ（初期案）

- フロントエンド: React / Next.js
- バックエンド: Node.js / TypeScript
- DB: PostgreSQL
- 画像保存: S3互換ストレージ等
- 将来連携: Shopify Admin API

---

## 16. 追加で決めたい事項

1. 練習用アプリで完結するか、Shopify開発ストアと実際に連携するか
2. ブランドページは「練習用プレビュー」のみか、「Shopifyテーマ反映」まで行うか
3. 在庫管理は単一倉庫か、複数ロケーション前提か
4. 商品登録は手入力中心か、CSV取込も初期から入れるか
5. 学習用途だけでなく、社内研修評価機能まで持たせるか
